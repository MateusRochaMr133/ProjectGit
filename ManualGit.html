O que é o Git

O Git é um projeto de código aberto maduro e com manutenção ativa desenvolvido em 2005 por Linus Torvalds, o famoso criador do kernel do sistema operacional Linux. Um número impressionante de projetos de software depende do Git para controle de versão, incluindo projetos comerciais e de código-fonte aberto.


O que é Controle de Versão?

É o acompanhamento e gerenciamento das mudanças feitas no código ao longo do tempo, tornando-as rastreáveis e gerenciáveis. É essencial fazer o controle da versão para evitar e proteger contra possíveis erros e bugs futuros no código.

Formas de resguardar contra bugs

  1. Controle de Versão: Permite rastrear todas as alterações feitas no código
  2. Registros Temporais: Marcam o momento em que as mudanças foram feitas, ajudando na identificação de problemas
  3. Gerenciamento Administrativo: Define responsabilidades e processos para alterações no código.
  4. Comentários (Commits): Descrevem as alterações feitas em detalhes, facilitando a compreensão.
  5. Revertendo para Estados Anteriores: Possibilita retornar o código a versões anteriores para corrigir problemas.

O que é um Repositório no Git?

É um diretório onde os arquivos do seu projeto ficam armazenados. Ele pode ficar em um depósito do GitHub ou em seu computador. Você pode armazenar códigos, imagens, áudios, ou qualquer outra coisa relacionada ao projeto no diretório.

Comando Básicos do Git

A sensibilidade a maiúsculas e minúsculas é geralmente importante. No Git Bash, os comandos são case-sensitive, o que significa que "ls" e "LS" são tratados como comandos diferentes.

Portanto:

ls listará arquivos e diretórios no diretório atual em letras minúsculas

LS não será reconhecido como um comando válido e resultará em um erro em alguns casos de uso.

Comando CD

Comando para mudar o diretório de um local de origem para outro local desejado. Para acessar subdiretórios você pode usar “/”. Os susbdiretórios são listados em ordem crescente.

Git Init

comando git init é usado para criar um novo repositório Git em um diretório vazio ou em um diretório existente, tornando-o um repositório Git.Este comando é usado uma única vez para inicializar um novo projeto ou para converter um diretório existente em um repositório Git.

Git Branch

comando git branch é usado para listar, criar, renomear ou excluir ramificações (branches) em um repositório Git.Sem argumentos, ele lista todas as ramificações existentes no repositório. Com argumentos, ele permite realizar várias operações relacionadas a ramificações.

O comando git config --global user.name mateus.rocha

É usado para configurar o nome de usuário global do Git em um sistema. Essa configuração é importante porque o Git registra informações sobre o autor de cada commit que é feito em um repositório. Portanto, configurar o nome de usuário é uma maneira de identificar quem fez as alterações em um repositório.

O comando git config --global user.email mateusrochamr@outlook.com

É usado para configurar o endereço de e-mail global do Git em um sistema. Assim como o nome de usuário, configurar o endereço de e-mail é importante porque o Git usa essa informação para registrar o autor de cada commit em um repositório. Aqui está a explicação detalhada deste comando:
